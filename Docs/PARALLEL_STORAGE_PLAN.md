**Features Implemented:**
- ? `ParallelMode` enum: None, Auto, Buffered, Latched, Optimistic
- ? `ParallelModeOptions` with presets: Default, HighWriteThroughput, MixedWorkload, Debug
- ? Builder methods: `WithParallelWrites()`, `WithMaxWriters()`, `WithoutParallelWrites()`
- ? Connection string: `Parallel Mode=Auto;Max Writers=8`
- ? Full validation and round-trip parsing

**API Usage:**
```csharp
// Builder API
var db = new WitDatabaseBuilder()
    .WithFilePath("db.witdb")
    .WithBTree()
    .WithParallelWrites(ParallelMode.Auto)
    .WithMaxWriters(8)
    .Build();

// Or with preset
var db = new WitDatabaseBuilder()
    .WithFilePath("db.witdb")
    .WithLsmTree()
    .WithParallelWrites(ParallelModeOptions.HighWriteThroughput)
    .Build();

// Connection string
var cs = "Data Source=db.witdb;Store=btree;Parallel Mode=Auto;Max Writers=8";
using var conn = new WitDbConnection(cs);
```

## Decision Log

| Date | Decision | Rationale |
|------|----------|-----------|
| 2025-06-11 | Start with WAL-based approach | Simpler, proven pattern |
| 2025-06-11 | Use System.Threading.Channels | Lock-free, high performance |
| 2025-06-11 | Use System.Threading.Lock | .NET 9+ optimized lock type |
| 2025-06-11 | Thread-local buffers for LSM | Reduce contention on MemTable |
| 2025-06-11 | Page-level latching for BTree | Fine-grained concurrency |
| 2025-06-11 | ReaderWriterLockSlim for latches | Supports shared/exclusive modes |
| 2025-06-11 | Optimistic reads by default | Avoid latch overhead for reads |
| 2025-06-11 | ParallelMode enum | Clear, type-safe configuration |
| 2025-06-11 | Presets for common scenarios | Easy configuration |
